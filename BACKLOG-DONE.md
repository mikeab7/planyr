### B65 — White flashing on zoom/pan and general redraw `[Site Planner / map]` (bug)
`[x]` **Done 2026-06-19 (branch `claude/nice-heisenberg-r6lmf1`).** Zooming/panning a located site flashed (originally white) as the aerial blanked between redraws. **Root cause, measured not guessed** (headless probe `ui-audit/diagnose-flash.mjs`): the basemap view-lock effect called `map.setView(center, z, {animate:false})` on *every* planner `view` change; Leaflet's `setView`→`_resetView` fires `viewreset`, whose GridLayer handler `_resetAll` **wipes and reloads all tiles**, so each wheel step blanked the aerial for a frame (loaded tiles collapsed **22 → 2**), exposing the backdrop.
> **Fix shipped (two parts):**
> 1. **Mitigation (commit `cf26912`):** the basemap container backdrop was `PAL.paper` (`#f4f1ea`, near-white) → any gap read as a *bright* flash. Changed to a neutral mid-dark gray (`#3f3f3f`) **only while the aerial is on** (stays `PAL.paper` with the aerial off so the SVG-off background still matches). Bright flash → dim blink.
> 2. **Real fix — seamless transform-based zoom (commit `af10d1c`):** stop re-`setView`-ing on every frame. Hold Leaflet at a committed center/zoom and, for the small deltas of a live gesture, CSS-`transform` (translate+scale) the whole map container so the already-loaded tiles (and the shared FEMA/NWI overlay layers, which live on the same map) scale to track the gesture — no tile reload, no wipe, no flash. A debounced (180 ms) `setView` re-renders crisp once the gesture settles. First paint / resize / >1.5-zoom-level jumps commit immediately so the aerial is never scaled into a blurry mess. Transform math: `scale = getZoomScale(z, committed)`; translate so the desired center maps to the container centre (matches the SVG, which renders at true scale every frame), `transform-origin:0 0`. Container CSS transforms don't affect Leaflet's coordinate math (it uses cached `_size`/pixel-origin, and `clientWidth` is layout-box, unaffected by transform), so alignment is preserved.
> **Tested + rejected on the way:** flipping the map to `zoomAnimation/fadeAnimation:true` made it *worse* (gap frames 1→4, min loaded still 2) and reintroduces SVG-desync risk — not used.
> **Verified headless:** `diagnose-flash.mjs` now reports **0 gap frames** (min loaded 8, was 2); `ui-audit/verify-zoom-align.mjs` confirms the drawn building/parcel stay locked to the aerial through the transform and after the crisp commit (no jump). lint 0 errors · 257 tests · build green. **Live confirmation on planyr.io still nice-to-have** (a one-frame flash can't be captured headless), but the mechanism + alignment are proven. **Note:** Document Review's PDF viewer uses a different (canvas raster) renderer, not Leaflet — if zoom-flashing shows there too it's a separate fix, tracked if/when reported.

### B128 — Split tool: correctly cut concave parcels (general multi-piece straight cut) `[Site Planner]` (bug → enhancement)  *(minted B127 on-branch; renumbered B128 after `main`'s #85 concurrently took B127 for the rail/header dropdown fix — the implementing commit still says "B127")*
`[x]` Done 2026-06-17 (branch `claude/inspiring-gates-qrhziq`). Closes the QA marathon's last "substantive bug still present" hand-off and supersedes the long-deferred **B31**.
- **The data-corruption fear was already mitigated, and that was verified, not assumed.** The hand-off carried an `[S2/HIGH]` "Split corrupts area on concave parcels — saved silently, no error." That premise was **stale** (written against pre-#48 code): `performSplit` already had a concave-cut **backstop guard** (area-conservation + per-piece self-intersection check, from the B74–B92 batch) that *refuses* a bad cut with a warning instead of saving corrupted geometry. Confirmed by replicating the live `splitPolygon` + guard and fuzzing **320k random concave cuts** (combs/L/U/stars) against a Monte-Carlo point-membership oracle: **zero** wrong splits slipped past the guard; every 4/6/8/10/12-crossing cut was rejected. So there was no silent `siteSqft`/FAR/coverage corruption to fix.
- **The real remaining gap was capability, not safety.** Because the old `splitPolygon` only paired the two extreme-`t` crossings, a straight cut that crosses a concave lot **4+ times** (which honestly yields **k+1** pieces for `2k` crossings — e.g. 3 for a U cut across both prongs) could only ever produce 2 overlapping/omitting "halves," so the guard always **refused** it. Net user-visible behavior: "the Split tool won't cut an L/U/notched lot through its reentrant part."
- **Fix — a correct general slice.** Extracted the parcel-split geometry out of `SitePlanner.jsx` into a new pure, tested module **`lib/polygonSplit.js`**: `splitPolygonByLine(points,A,B)` classifies each vertex to a binary side (guarantees an even crossing count — no odd-parity tangency bug), inserts a crossing node on every sign-change edge, **pairs ALL crossings even–odd along the cut line** (each pair is an interior chord), and traces every resulting piece (walk a boundary arc → hop the interior chord to its partner → continue), dropping zero-area slivers. Returns the natural set of pieces (`>=2`) or `null`. The bent-polyline cut moved over too as `splitPolygonByPath` (unchanged logic). `performSplit` now replaces the one parcel with **N** parcels and keeps the area/self-intersection guard as a backstop, so the result can never be worse than before (a rare near-vertex-grazing degenerate sliver still self-intersects → still refused).
- **UX change to note:** a straight cut through a notched lot now creates **all** the real pieces (e.g. 3 parcels for a U-shaped lot), instead of declining. Convex and 2-crossing concave cuts are byte-for-byte the same two pieces as before.
- **Validation:** new `test/polygonSplit.test.js` (11 cases: convex halving, L 2-crossing, **U → 3 pieces with exact areas + membership**, 3-tooth comb → 4 pieces, line-misses/degenerate → null, bent-path cut, and a 1,500-cut randomized area+membership property check). Standalone prototype passed the 300k-trial oracle (0 area / 0 membership failures; ~0.003% near-degenerate self-int → safely refused). Full suite **163 tests** (was 152, +11) · lint **0 errors** · build green · lazy-chunk split intact. Owner approved shipping; merged to `main` via PR.
